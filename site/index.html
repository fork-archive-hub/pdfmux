pdfmux
docs pypi github

PDF extraction that
checks its own work

Extracts every page. Audits quality. Re-extracts the ones that came out wrong. Python API + CLI + MCP server.

get started → v0.4.0 on PyPI
pip install pdfmux copy

Python API

import pdfmux

# extract as markdown
text = pdfmux.extract_text("report.pdf")

# structured json with locked schema
data = pdfmux.extract_json("report.pdf")

# LLM-ready chunks with token estimates
chunks = pdfmux.load_llm_context("report.pdf")
# → [{title, text, page_start, page_end, tokens, confidence}]

CLI

# convert — auto-audits every page
$ pdfmux invoice.pdf
✓ invoice.md (2 pages, 95% confidence)

# image-heavy pdf — bad pages re-extracted with OCR
$ pdfmux pitch-deck.pdf
✓ pitch-deck.md (12 pages, 85% confidence, 6 OCR'd)

# LLM-ready chunked json
$ pdfmux report.pdf -f llm

# per-page extraction breakdown
$ pdfmux analyze report.pdf

# start MCP server for AI agents
$ pdfmux serve

How it works

1. Fast extract — PyMuPDF on every page (instant)
2. Audit — classify each page: good / bad / empty
3. Re-extract — OCR only the bad pages
4. Merge — combine good + fixed pages in order
All pages good? Zero OCR overhead. You only pay for what's broken.

Features

Multi-pass pipeline
Extract, audit, re-extract. Every page verified.
LLM chunking
Section-aware splitting with token estimates for RAG.
Honest confidence
0-100% score that reflects actual extraction quality.
MCP server
Give AI agents the ability to read any PDF.
4 output formats
Markdown, JSON, CSV, or LLM-chunked JSON.
Zero config
Works out of the box. OCR only when you need it.

Optional extras

# add table extraction
$ pip install "pdfmux[tables]"

# add Gemini Flash for complex layouts
$ pip install "pdfmux[llm]"

# install everything
$ pip install "pdfmux[all]"
